# Business Rules: Nexus / Portafolio

## Módulo: Core
- Todas las funciones Python deben implementar `try-except` para robustez.
- Separación estricta entre lógica (Python) y presentación (Jinja2).

## Módulo: Git
- Los cambios críticos deben ser validados contra el remoto.
- Usar checkpoints frecuentes.

## Módulo: Assets & UI
- Estructura obligatoria en /assets para CSS/JS externos.
- Prohibición absoluta de estilos inline (style="") y bloques de estilo internos.
- Uso mandatorio de tokens.css para variables de diseño.
